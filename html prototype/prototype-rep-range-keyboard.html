CURRENT

Reps field carries too much

The active reps field can act like target entry and performed reps entry, so target edits and actual logging compete for the same space.

PROPOSED

Target gets its own control

The reps field stays single-purpose. Target editing moves into the keyboard toolbar with exact and range modes plus explicit Apply and Clear actions.

CONTINUITY

Template to workout stays visible

The template source, current target, engine-chosen reps, and actual performed reps stay visible together so it is easier to reason about what the set is asking for.
