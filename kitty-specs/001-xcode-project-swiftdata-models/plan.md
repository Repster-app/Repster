**SwiftData-specific notes:**
- Enums used in @Model must conform to `Codable` and `String` (raw value) for SwiftData storage.
- Optional enums (like `cachedPRStatus: CachedPRStatus?`) work natively in SwiftData.
- Arrays of Codable types (`[String]` for secondaryMuscles) are stored as transformable.
- `@Attribute(.unique)` can enforce uniqueness on PerformanceRecord's composite key, but SwiftData doesn't natively support multi-column unique constraints. Enforce in repository/service layer instead.
- Use `@Relationship` for inverse relationships where needed (e.g., Workout has many WorkoutSets).
- Lightweight migration handles additive schema changes automatically.

```swift
var hasData: Bool {
    ((weight ?? 0) > 0 && (reps ?? 0) > 0) ||
    (durationSeconds ?? 0) > 0 ||

var volume: Double? {
    guard let ew = effectiveWeight, let r = reps, r > 0 else { return nil }
    return ew * Double(r)
}
```

## Implementation Approach

### Order of Implementation

1. **Enums first** - all 9 enum files (no dependencies)
2. **Models second** - all 11 @Model classes (depend on enums)
3. **ModelContainer config** - wire up in App struct
4. **File structure** - create all directory scaffolding with placeholder files
5. **App entry point** - ReppoApp.swift with ContentView placeholder
6. **Verify build** - zero errors, zero warnings

### Key Decisions

- **No @Relationship annotations in v1**: Use UUID foreign keys (workoutId, exerciseId, etc.) rather than SwiftData @Relationship. This keeps models simple and avoids cascade-delete complexity in SwiftData. Relationships are navigated via repository queries. Can add @Relationship later if needed.
- **No indexes in this feature**: Database indexes (feature 002) require the repository layer. ModelContainer setup here is basic.
- **Defaults in initializers**: Use Swift default parameter values in init() for common defaults (completed: false, setType: .working, etc.)
- **Xcode project creation**: Use `swift package init` or create manually via Xcode. The .xcodeproj must be committed.
