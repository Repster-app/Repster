## Context & Constraints

**Threading model**: Every repository is a `@ModelActor` actor. The macro auto-generates:
- `nonisolated let modelExecutor: any SwiftData.ModelExecutor`
- `nonisolated let modelContainer: SwiftData.ModelContainer`
- `init(modelContainer: SwiftData.ModelContainer)` which creates a fresh `ModelContext` and wraps it in `DefaultSerialModelExecutor`

**Key rules**:
- Methods inside the actor are synchronous (they run on the actor's serial executor)
- The protocol declares `async throws`; implementations should use just `throws`
- `modelContext` is available directly inside the actor (generated by `@ModelActor`)
- After every mutation (insert/delete), call `try modelContext.save()`
- Use `FetchDescriptor<T>` with `#Predicate` for all queries
- Use `SortDescriptor` for ordering
- Use `descriptor.fetchLimit` to constrain result count
- Return `@Model` objects directly (safe within the same actor context)

**SwiftData limitations**:
- No native SUM/AVG/GROUP BY — no `fetchTotalVolume` on SetRepository; callers read `ExerciseStats.totalVolume` (pre-computed at write-time)
- `#Predicate` with optional fields requires careful handling (use `$0.reps == reps` where reps is non-optional, or explicit nil checks)
- No compound indexes on iOS 17 — queries work correctly but without index optimization

**Reference documents**:
- `kitty-specs/002-repositories-and-indexes/plan.md` — Implementation pattern with code examples
- `kitty-specs/002-repositories-and-indexes/data-model.md` — Detailed method specifications per repository
- `kitty-specs/002-repositories-and-indexes/research.md` — SwiftData aggregation strategy
- `kitty-specs/002-repositories-and-indexes/contracts/` — Protocol signatures

func fetch(exerciseId: UUID, recordType: RecordType, reps: Int?) throws -> PerformanceRecord? {
    let descriptor: FetchDescriptor<PerformanceRecord>

    if let reps {
        descriptor = FetchDescriptor<PerformanceRecord>(
            predicate: #Predicate {
                $0.exerciseId == exerciseId
                && $0.recordType == recordType
                && $0.reps == reps
            }

        descriptor = FetchDescriptor<PerformanceRecord>(
            predicate: #Predicate {

Test this during implementation.

**Closest-weight lookup (FR-008)** — two-query algorithm:
```swift
func fetchClosest(to date: Date, healthProfileId: UUID) throws -> BodyweightEntry? {
    // Query 1: Nearest entry BEFORE or ON the target date
    var beforeDescriptor = FetchDescriptor<BodyweightEntry>(
        predicate: #Predicate {
            $0.healthProfileId == healthProfileId && $0.date <= date
        },
        sortBy: [SortDescriptor(\.date, order: .reverse)]
    )
    beforeDescriptor.fetchLimit = 1
    let before = try modelContext.fetch(beforeDescriptor).first

    // Query 2: Nearest entry AFTER the target date
    var afterDescriptor = FetchDescriptor<BodyweightEntry>(
        predicate: #Predicate {
            $0.healthProfileId == healthProfileId && $0.date > date
        },
        sortBy: [SortDescriptor(\.date, order: .forward)]
    )
    afterDescriptor.fetchLimit = 1
    let after = try modelContext.fetch(afterDescriptor).first

    // Return whichever is closer (by absolute time distance)
    switch (before, after) {
    case let (b?, a?):
        let beforeDistance = date.timeIntervalSince(b.date)
        let afterDistance = a.date.timeIntervalSince(date)
        return beforeDistance <= afterDistance ? b : a
    case let (b?, nil):
        return b
    case let (nil, a?):
        return a
    case (nil, nil):
        return nil
    }
}
```

**Files**: `Reppo/Core/Repositories/BodyweightEntryRepository.swift` (new, ~60 lines)
**Parallel?**: Yes (independent file)

**Edge cases**:
- No bodyweight entries at all → return nil. Caller (SetService) falls back to `effectiveWeight = weight` with user warning (AGENT_RULES S3.3).
- Exactly one entry → that entry is always closest.
- Entry on exact same date → returned by the "before" query (uses `<=`).
- Ties (equal distance before and after) → prefer the earlier entry (consistent with "earliest wins" principle).

## Risks & Mitigations

| Risk | Severity | Mitigation |
|------|----------|-----------|
| `#Predicate` fails with optional field comparisons (`reps: Int?`) | Medium | Test compilation; use explicit nil checks or unwrap patterns in predicate |
| `#Predicate` fails with enum comparisons (`RecordType`) | Medium | Fall back to raw value comparison: `$0.recordType.rawValue == recordType.rawValue` |
| `volume` computed property not accessible after fetch | Low | SwiftData @Model computed properties should be accessible. If not, compute `(effectiveWeight ?? 0) * Double(reps ?? 0)` inline |
